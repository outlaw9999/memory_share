# AMP-0001: Architecture Memory Protocol (Specification)

**Status:** Draft / Experimental
**Version:** 1.3.1
**Author:** Architecture Guardian / .kit Team

---

## 1. Abstract

The Architecture Memory Protocol (AMP) defines a deterministic, machine-readable interface for AI agents to interact with software architecture. It eliminates "Blind Reading" of source code, replacing it with high-precision architectural facts (Assertions) served via a structured Memory Bus.

## 2. Terminology

*   **Symbol:** A named entity within the codebase (e.g., `Function`, `Class`, `Module`, `Decorator`).
*   **Assertion:** A deterministic fact extracted from AST or Git metadata (e.g., `A calls B`, `X inherits Y`).
*   **Intent:** A semantic explanation of a Symbol's purpose, stored in the Knowledge Layer.
*   **Memory Kernel:** The local engine responsible for indexing, storage (`atlas_v1.db`), and query resolution.
*   **Memory Bus:** The transport layer (CLI/JSON-RPC) that mediates between the Kernel and the Agent.
*   **Reasoning Contract:** The JSON schema governing the communication between Kernel and Agent.

## 3. Protocol Architecture (Pull-Based)

AMP operates on a **Deterministic Pull** model. Agents must not guess; they must query.

### 3.1. Discovery Handshake
Every AMP-compliant kernel MUST support a capability discovery command.
**Command:** `kit capabilities --json`

```json
{
  "protocol": "kit.protocol.v1",
  "capabilities": ["why", "impact", "context", "doctor", "batch"],
  "kernel_version": "1.3.1",
  "engine": "python-ast-v1",
  "status": "ready"
}
```

### 3.2. Reasoning Contract (v1)
The standard output for a `why` query.
**Command:** `kit why <Symbol_Name> --json`

```json
{
  "schema": "kit.reasoning.v1",
  "metadata": {
    "commit": "abc1234",
    "timestamp": "2026-03-14T08:00:00Z",
    "stale": false
  },
  "symbol": {
    "name": "ApplicationService.index_codebase",
    "type": "method",
    "intent": "Scans and indexes the codebase using AST parsing."
  },
  "structural": {
    "callers": ["cli.index_cmd"],
    "callees": ["GraphStore.update_file"],
    "dependencies": ["os", "pathlib", "sqlite3"]
  },
  "constraints": ["No background-sync", "Thread-safe"]
}
```

## 4. Error Model

AMP forbids ambiguous errors. All exceptions must return a `kit.error.v1` schema.

| Code | Meaning | Recovery Action |
| :--- | :--- | :--- |
| `SYMBOL_NOT_FOUND` | Symbol does not exist in graph | Check spelling or run partial index |
| `STALE_INDEX` | Code changed; graph is outdated | Run `kit index --incremental` |
| `SCHEMA_MISMATCH` | Agent/Kernel version conflict | Upgrade `kit` or downgrade Agent |

**Example Error:**
```json
{
  "schema": "kit.error.v1",
  "error": {
    "code": "STALE_INDEX",
    "message": "Index is 5 commits behind current HEAD.",
    "recovery": "kit index"
  }
}
```

## 5. Engineering Doctrine (AMP-SIG)

Implementations of this protocol (like `.memory_share.kit`) MUST adhere to:
*   **Correctness-First:** A fact is either 100% accurate or non-existent (return Error).
*   **Token Efficiency:** Responses must be compressed for LLM context windows.
*   **Zero-Overhead:** No background processes or hidden telemetry allowed (Architecture Martial Law).
