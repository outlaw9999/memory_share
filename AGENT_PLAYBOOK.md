# 🛡️ AGENT PLAYBOOK (Governance & Anti-Loop)

This document defines the constraints and operating procedures for all AI agents working in this repository. 

## 🚫 ANTI-LOOP POLICY (The Checkpoint Rule)
To prevent resource depletion and "Ghost Worker" syndrome, the following rules are mandatory:

1. **Max Iterations**: Agents are FORBIDDEN from running autonomous loops for more than **5 iterations** on the same problem.
2. **Mandatory Checkpoint**: If a task requires more than 5 attempts (e.g., fixing a persistent test failure or build error), you MUST stop immediately.
3. **Cognitive Snapshot**: Before stopping, run `kit learn --tag decision "Failed to resolve [X] due to [Y]"` to document the effort.
4. **Human Intervention**: Notify the human user and request intervention.
5. **No Silent Loops**: Running in the background without periodically updating the memory (`kit learn` or `kit commit`) for more than 30 minutes is considered a violation of governance.

## 📖 CONTEXT AUTHORITY
- **Canonical Source**: The `.kit/context` and `AGENTS.md` files (generated by `.kit`) are the **absolute source of truth**.
- **Conflict Resolution**: If `README.md` or other narrative documents conflict with `.kit` manifests, you MUST prioritize `.kit`.

## 🧠 EPISTEMIC AUTHORITY
- **Humans Only**: Only human users have the authority to define immutable "Semantic" facts. 
- **Drift Detected**: If Vantage signals a structural change, and it conflicts with known architectural facts, identify it as `DRIFT` and request human reconciliation (`kit learn --supersede`).
